# Font Unifier - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
Font Unifier

### 1.2 产品描述
Font Unifier 是一个基于 Python 的桌面应用程序，用于批量统一 Microsoft Office 文件（Word、Excel、PowerPoint）的字体。该工具通过图形用户界面（GUI）让用户轻松选择文件并指定目标字体，实现字体统一化处理。

### 1.3 产品目标
- 简化 Office 文件的字体管理
- 提高办公效率，减少手动调整字体的时间
- 提供直观易用的界面，适合非技术用户

### 1.4 核心功能
- 支持 .docx、.xlsx、.pptx 文件格式
- 批量更改文档中所有文本的字体
- 图形用户界面操作
- 自动保存修改后的文件

## 2. 功能需求

### 2.1 文件选择功能
- 用户可以通过浏览按钮选择单个 Office 文件
- 支持文件类型过滤：.docx、.xlsx、.pptx
- 显示选定文件的完整路径

### 2.2 字体设置功能
- 用户可通过下拉框选择目标字体，也可直接输入
- 下拉列表枚举系统已安装的全部字体（`QFontDatabase.families()`）
- 输入时按前缀（大小写不敏感）自动匹配候选字体并弹出列表
- 点击字体框任意位置即可弹出候选列表（行为对标 Excel）
- 默认字体设置为 "Meiryo UI"

### 2.3 文件处理功能
- **Word 文件处理**：
  - 更改所有段落文本的字体
  - 更改表格中所有单元格文本的字体
  - 同时设置西文与东亚（eastAsia）字体属性，确保中日文字符生效
  - 清除 `asciiTheme/hAnsiTheme/eastAsiaTheme/cstheme` 等主题引用，避免残留导致回退到主题字体
- **Excel 文件处理**：
  - 遍历 `workbook._fonts` 全部字体定义，替换字体名并把 `scheme` 置 `None`
  - 一次覆盖所有工作表的全部单元格、命名样式与默认（Normal）字体
  - 保留原有字号、粗体、颜色等样式
- **PowerPoint 文件处理**：
  - 更改所有幻灯片中形状（文本框）文本的字体
  - 处理表格、图表（标题与坐标轴标题）、组形状（递归）
  - 处理过程对单形状异常进行兜底，避免单一图表异常导致整个文件失败

### 2.4 输出功能
- 自动生成修改后的文件
- 文件命名规则：原文件名 + "_modified" + 扩展名
- 保存到原文件所在目录

### 2.5 用户界面功能
- 浅色现代风界面：卡片式布局、靛蓝强调色、圆角
- 文件类型徽标（.docx / .xlsx / .pptx 三色标签）
- 处理中显示加载动画（busy `QProgressBar`），按钮置灰防重复触发
- 状态文本用圆角色块显示（成功=绿 / 错误=红 / 信息=靛蓝）
- 错误提示和成功确认对话框

## 3. 非功能需求

### 3.1 性能需求
- 处理速度：单个文件处理时间不超过 30 秒
- 内存使用：处理大文件时不超过 500MB
- 界面响应：文件处理在后台线程（QThread）执行，主界面不冻结，处理中可显示状态并禁用重复触发

### 3.2 兼容性需求
- 支持 Windows 操作系统
- 兼容 Python 3.12+
- 支持 Microsoft Office 文件格式

### 3.3 可靠性需求
- 错误处理：完善的异常捕获和用户提示
- 数据安全：不修改原文件，只生成新文件

### 3.4 用户体验需求
- 界面响应迅速
- 操作步骤简单明了
- 提供清晰的状态反馈

## 4. 技术实现

### 4.1 技术栈
- **编程语言**：Python 3.12+
- **GUI 框架**：PyQt6
- **Office 文件处理**：
  - Word：python-docx
  - Excel：openpyxl
  - PowerPoint：python-pptx

### 4.2 架构设计
- 核心逻辑层：字体更改函数（含东亚字体属性处理）
- 界面层：PyQt6 GUI 应用
- 文件处理层：Office 文件读写操作
- 后台处理层：QThread worker，避免阻塞主界面
